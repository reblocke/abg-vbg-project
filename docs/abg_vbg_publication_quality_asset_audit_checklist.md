# ABG/VBG Manuscript Asset Publication-Quality Audit Checklist

## Purpose

Use this checklist to audit every manuscript and supplement figure/table generated by the ABG/VBG Quarto workflow. The goal is to ensure that each asset is:

1. methodologically correct,
2. publication-ready,
3. self-explanatory,
4. visually readable,
5. synchronized with manuscript text,
6. free of stale/pilot/debug artifacts,
7. and correctly represented in the asset registry.

This checklist is intended for manual review and for Codex-assisted automated QA.

---

## Severity scale

Use this severity scale for every audit item.

| Severity | Meaning | Action |
|---|---|---|
| **Fatal** | Asset is missing, mislabeled, stale, numerically wrong, or contradicts manuscript claims | Must fix before circulation |
| **Major** | Asset is present but not publication-ready, difficult to read, incompletely captioned, or methodologically ambiguous | Fix before coauthor/journal submission |
| **Minor** | Cosmetic or style issue that does not affect interpretation | Fix before final submission |
| **Pass** | Meets criterion | No action |

---

# I. Universal figure/table requirements

Apply to every figure and table.

## A. Identity, naming, and placement

- [ ] Asset has the correct manuscript label: Figure 1, Figure 2, Table 1, Table 2, Figure S1, Table S1, etc.
- [ ] Asset label matches the manuscript text, caption, registry, and output filename.
- [ ] Asset appears only once in the manuscript-facing render unless explicitly repeated for QA.
- [ ] Deprecated/draft-only versions are not printed as active manuscript assets.
- [ ] The figure/table appears in the expected order.
- [ ] Main-text figures/tables are not mixed with supplement-only displays.
- [ ] Supplement numbering is stable and consistent.
- [ ] No stale pilot/test output is being used as a manuscript-facing asset unless explicitly labeled as pilot validation.

## B. Standalone interpretability

- [ ] Title/caption states what is shown.
- [ ] Caption identifies the population/cohort analyzed.
- [ ] Caption identifies the analytic method, if needed.
- [ ] Caption defines abbreviations: ABG, VBG, pCO2, IPSW, MI, GBM, OR, CI, NIV, IMV, HCRF.
- [ ] Caption defines reference groups and reference values.
- [ ] Caption states whether percentages are column percentages, row percentages, weighted percentages, or predicted probabilities.
- [ ] Caption states whether estimates are crude, adjusted, weighted, MI-pooled, or sensitivity analyses.
- [ ] Asset can be understood without reading the surrounding text.

## C. Consistency with manuscript

- [ ] Figure/table supports a result explicitly mentioned in Results, Methods, or Supplement.
- [ ] Manuscript text does not describe a different analysis than the asset shows.
- [ ] Numerical values in text match the table/figure.
- [ ] The same terminology is used across manuscript, caption, and table/figure.
- [ ] The same outcome order is used across assets unless there is a clear reason to change it.
- [ ] The same modality order is used across assets: preferably ABG then VBG.
- [ ] The same category order is used across assets: Low, Normal, High.
- [ ] Reference categories are clearly stated and consistent.

## D. Numerical communication

- [ ] Absolute numbers are provided when percentages are reported.
- [ ] Relative measures are paired with 95% confidence intervals.
- [ ] Predicted probabilities or absolute risks are included where they help clinical interpretation.
- [ ] Denominators are clear and consistent.
- [ ] Avoid "1-in-X" denominators unless specifically needed.
- [ ] Do not mix relative and absolute formats in ways that exaggerate effects.
- [ ] Decimal precision is consistent and meaningful.
- [ ] P values are not the sole evidence presented.
- [ ] Avoid verbal-only probability descriptions such as "common" or "rare" without numbers.

## E. Publication-readiness

- [ ] No raw/debug/internal columns are visible.
- [ ] No code, helper object names, file paths, or chunk names appear in manuscript-facing assets.
- [ ] No clipped captions, clipped legends, clipped axes, or truncated footnotes.
- [ ] Font sizes are readable at final journal scale.
- [ ] Labels remain readable in grayscale.
- [ ] Colors are colorblind-accessible.
- [ ] Lines/points/CI ribbons are distinguishable in black-and-white or grayscale.
- [ ] No excessive gridlines, decorative styling, or redundant legends.
- [ ] Units appear in axis labels or table headers, not repeatedly in cells.
- [ ] Horizontal table lines are used sparingly; avoid vertical rules when possible.
- [ ] Wide tables are landscape, split into panels, or simplified.
- [ ] All output paths are deterministic.
- [ ] Asset is exported at appropriate resolution/dimensions for the target journal.

# II. Figure-specific audit criteria

## A. Layout and readability

- [ ] Panel arrangement matches analytic logic.
- [ ] Panel labels or facet labels are visible and consistent.
- [ ] Axes are shared when comparisons require shared scale.
- [ ] If axes differ, this is obvious and justified.
- [ ] Outcome order is clinically sensible and consistent.
- [ ] Modality mapping is consistent across panels.
- [ ] Legend is placed where it does not obscure data.
- [ ] Legend labels match manuscript terminology.
- [ ] Confidence intervals are visible but not visually dominant.
- [ ] Curves/points do not extend beyond support in a misleading way.

## B. Axis and scale checks

- [ ] Axis labels include units.
- [ ] Log scales are explicitly labeled.
- [ ] Odds ratio figures include a visible null/reference line at OR = 1.
- [ ] Probability figures use percentages or probabilities consistently.
- [ ] Probability axes start at 0 when appropriate.
- [ ] Probability displays do not visually exaggerate small differences.
- [ ] Reference values are stated for OR curves.
- [ ] Displayed range is stated if trimmed, e.g., 2nd-98th percentile.
- [ ] CO2 labels are safe and consistent: prefer `pCO2`, `PaCO2`, `PvCO2` or a tested glyph-safe format.

## C. Methodological clarity

- [ ] Figure specifies whether curves are crude, adjusted, weighted, MI-pooled, or sensitivity analyses.
- [ ] Figure specifies whether predictions are reference-profile predictions or marginally standardized predictions.
- [ ] If probabilities are shown, caption states how they were generated.
- [ ] If weights are used, caption states what population the weights target.
- [ ] If multiple imputation is used, caption states that results are MI-pooled.
- [ ] If sensitivity analysis is shown, caption states that it is a sensitivity analysis and not primary.

## D. Export checks

- [ ] Final asset is available as PDF/SVG or high-resolution PNG/TIFF, depending on journal needs.
- [ ] Raster figures are at least 300 dpi; 600-1000 dpi preferred for line-heavy plots.
- [ ] Figure dimensions are defined in physical units compatible with journal limits.
- [ ] Export settings are deterministic and not dependent on RStudio viewer state.
- [ ] SVG/PDF text and lines are not distorted when rendered.

# III. Table-specific audit criteria

## A. Layout

- [ ] Table title is specific and accurate.
- [ ] Column headers are short and interpretable.
- [ ] Row groups are meaningfully ordered.
- [ ] Total sample appears before subgroups.
- [ ] Comparisons proceed left-to-right.
- [ ] Units appear in row or column headers.
- [ ] Abbreviations are defined in footnotes.
- [ ] Table is not too wide for page.
- [ ] No detached columns or orphaned table fragments.
- [ ] No raw helper columns appear, e.g., `var_type`, `row_type`, `panel`, `analysis_family`, unless intended.

## B. Numeric formatting

- [ ] Continuous variables use mean +/- SD or median [IQR], consistently.
- [ ] Categorical variables use n (%).
- [ ] Percentages are clearly identified as row, column, weighted, or unweighted.
- [ ] Odds ratios are reported as OR (95% CI), or OR/LCL/UCL, but not both unless justified.
- [ ] Probability estimates are labeled as predicted probabilities, weighted probabilities, or observed proportions.
- [ ] Use consistent decimal places.
- [ ] Denominators are integers, not decimals.
- [ ] Missingness is not embedded in baseline table cells if there is a separate missingness table.
- [ ] Missing data are summarized in a dedicated table if MI is central.

## C. Statistical labeling

- [ ] Reference group is stated.
- [ ] Weighting method is stated if weighted.
- [ ] MI pooling is stated if MI-pooled.
- [ ] Sensitivity analyses are clearly labeled.
- [ ] Event counts and denominators are included where needed.
- [ ] If predicted probabilities are included, the estimation method is specified.
- [ ] If model outputs are conditional or marginal, that is stated.

# IV. ABG/VBG-specific expected display set

## Main paper

### Figure 1. Cohort assembly and target population for inverse probability-of-sampling weighting

Required checks:
- [ ] Shows extracted source cohort.
- [ ] Shows ambulatory exclusions.
- [ ] Shows analytic ED/inpatient cohort.
- [ ] Shows ABG-tested and VBG-tested cohorts.
- [ ] Indicates weighting target/common eligible source population.
- [ ] Does not imply mutually exclusive ABG/VBG groups if encounters can have both.
- [ ] Counts match the intended render mode.
- [ ] If full-cohort counts are intentionally shown in pilot mode, this is clearly documented.

### Table 1. Baseline characteristics of analytic cohort overall and by ABG/VBG receipt

Required checks:
- [ ] Includes overall cohort.
- [ ] Includes ABG yes/no columns.
- [ ] Includes VBG yes/no columns.
- [ ] Uses encounter-level language.
- [ ] Uses n (%) and mean +/- SD.
- [ ] Does not include embedded missingness strings.
- [ ] Does not include raw/internal fields.
- [ ] Includes footnote that patients may contribute multiple encounters, if applicable.
- [ ] Clarifies percentages are column percentages.
- [ ] Missingness is moved to Table S4.

### Figure 2. Primary weighted spline associations of pCO2 with clinical outcomes by ABG and VBG

Required checks:
- [ ] Shows ABG and VBG in parallel.
- [ ] Includes all four outcomes: IMV, NIV, 60-day mortality, hypercapnic respiratory failure diagnosis.
- [ ] Shows OR curves and predicted probability curves if both are claimed in text.
- [ ] OR curves include reference line at 1.
- [ ] Reference values are stated: ABG 40 mmHg, VBG 45 mmHg, if used.
- [ ] Predicted probabilities are described as marginally standardized or reference-profile, whichever is true.
- [ ] Confidence intervals/ribbons are visible.
- [ ] Display range is stated.
- [ ] No clipped caption or footnote.
- [ ] If spline-tail diagnostic warnings remain, figure is reviewed for visible instability.

### Table 2. Primary weighted categorical associations of pCO2 category with outcomes by ABG and VBG

Required checks:
- [ ] MI-logistic IPSW primary analysis only.
- [ ] Low vs Normal and High vs Normal contrasts.
- [ ] ABG and VBG paired by outcome.
- [ ] Includes OR (95% CI).
- [ ] Includes predicted probabilities if used in manuscript.
- [ ] Does not duplicate OR/LCL/UCL plus OR (95% CI) unnecessarily.
- [ ] No detached columns.
- [ ] Title and footnote state reference group and weighting/MI pooling.

# VI. Automated checks Codex should perform

## Asset registry checks
- [ ] Expected labels all present.
- [ ] No active duplicate labels.
- [ ] Deprecated labels suppressed.
- [ ] Artifact paths exist.
- [ ] Aliases point to current artifacts.

## PDF text checks
- [ ] Captions appear with correct labels.
- [ ] No "TBD," "placeholder," "draft only," or "Things like..." text in manuscript-facing sections.
- [ ] No unsupported Figure 3 if Figure 3 is deprecated.
- [ ] No "global SHAP" if figure is SHAP-style contribution.
- [ ] No "negative control outcome" language.
- [ ] No "pCO2" glyph corruption if using Unicode; otherwise use `pCO2`.

## Figure file checks
- [ ] Expected figure files exist.
- [ ] Image dimensions meet target.
- [ ] File sizes are not suspiciously tiny.
- [ ] No duplicated file hashes among canonical active figures.
- [ ] No clipped edges detected by bounding-box/canvas audit if feasible.

## Table checks
- [ ] No internal/debug columns.
- [ ] No detached/orphaned columns.
- [ ] No decimal denominators.
- [ ] No embedded missingness in Table 1.
- [ ] OR tables use one representation consistently.
- [ ] Percentages and denominators are clear.

## Numerical consistency checks
- [ ] Counts in Figure 1 match intended full-cohort flow manifest.
- [ ] Table 1 counts match analytic cohort for the render mode.
- [ ] Table 2 values match exported primary results CSV.
- [ ] Figure S4 values match Table 2.
- [ ] Table S2 values match crude results CSV.
- [ ] Table S3 values match GBM results CSV.
- [ ] Table S4 missingness values match pre-imputation missingness export.
- [ ] Table S5 values match MI diagnostic exports.

# VII. Poster-specific criteria

For the poster, the same assets may be reused, but the standards differ.

- [ ] Figures must be readable at poster viewing distance.
- [ ] Captions should be shorter than manuscript captions.
- [ ] Avoid dense tables on the poster.
- [ ] Prefer Figure 1, Figure 2, and a compact categorical summary.
- [ ] Use absolute probabilities or paired ABG/VBG estimates where space permits.
- [ ] Use plain-language labels: `Death by 60 days`, `Non-invasive ventilation`, etc.
- [ ] Do not include complex diagnostics unless the poster is methods-focused.
- [ ] Maintain same color/line mapping as manuscript.
- [ ] Avoid tiny footnotes; move details to QR code or supplement.

---

# VIII. Final pass checklist before coauthor circulation

- [ ] Main paper has no more than 2-3 main figures and 2 main tables unless journal allows more.
- [ ] Supplement has all diagnostics needed to support weighting, imputation, and sensitivity analyses.
- [ ] Main text and captions cite figures/tables in order.
- [ ] Every figure/table referred to in text exists.
- [ ] Every figure/table included is referred to in text.
- [ ] No stale pilot numbers appear in final manuscript-facing assets.
- [ ] Any pilot validation render is clearly labeled as pilot.
- [ ] All primary estimates in abstract, Results, Table 2, and Figure S4 agree.
- [ ] Captions are self-contained.
- [ ] Tables and figures use consistent terminology.
- [ ] Display set supports the manuscript story without redundant clutter.
